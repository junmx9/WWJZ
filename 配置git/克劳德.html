      • 远程 GitHub 仓库

      • 使用git clone命令来自 GitHub 克隆代码
      • 之后可以通过git pull更新代码

    我来帮您设置 GitHub Actions 自动部署流程。这需要几个关键步骤：

    1. 在 GitHub 仓库中创建部署密钥

    首先需要在服务器上生成SSH密钥对：

    狂欢
    ssh-keygen -t rsa -b 4096 -C "your-email@example.com"
    1.  配置 GitHub 秘密

    在 GitHub 仓库中:

    • 进入“设置” > “秘密和变量” > “操作”
    • 添加以下秘密：
      • SERVER_HOST: 您的服务器的IP地址
      • SERVER_USERNAME: 服务器
      • SERVER_SSH_KEY: 刚才身高的私钥内容

      • DEPLOY_PATH ：安装路径（例如 /opt/project/server-connection）
    1. 创建 GitHub Actions 工作流配置

    在您的仓库中创建.github/workflows/deploy.yml文件：

    yaml
    name: Deploy to Server

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Setup SSH
        run: |
          mkdir -p ~/.ssh
          echo "${{ secrets.SERVER_SSH_KEY }}" > ~/.ssh/deploy_key
          chmod 600 ~/.ssh/deploy_key
          
      - name: Deploy to Server
        run: |
          ssh -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no ${{ secrets.SERVER_USERNAME }}@${{ secrets.SERVER_HOST }} "
            cd ${{ secrets.DEPLOY_PATH }} &&
            git pull origin main
          "

    这个配置文件的作用是：

    • 监听主分支的个体事件
    • 使用配置的 SSH 连接到服务器
    • 自动执行 git pull 更新代码

    完成这些配置后，只要您的自动化代码到 GitHub 的主分支时，GitHub Actions 就会自动将更新部署到您的服务器。
